# studio-web-ui（变更增量）

## ADDED Requirements

### Requirement: 叠层控件点击不穿透到画布
当 HTML 控件叠在 Phaser 画布之上时，用户 SHALL 能在该控件可见区域内完成点击/拖拽，而不误触发下层画布平移或缩放，除非该区域明确设计为透明穿透。

#### Scenario: 左侧顶栏状态区
- **WHEN** 用户点击 HUD 顶部标题行或状态药丸所在区域
- **THEN** 浏览器 SHALL 命中该 UI 层，而不应触发画布拖拽起始（在常规桌面浏览器下）

#### Scenario: 抽屉打开时
- **WHEN** 用户点击抽屉遮罩或抽屉面板内容
- **THEN** 事件 SHALL 由遮罩/面板处理，而不应穿透到画布交互

#### Scenario: 拖拽画布后不误点选
- **WHEN** 用户拖拽平移画布并在工位或房间热点上松开指针，且移动距离超过约 12 像素
- **THEN** 系统 SHALL NOT 将该次操作视为对工位或房间的点击选中
