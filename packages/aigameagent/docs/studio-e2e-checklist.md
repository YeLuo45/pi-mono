## 9. 固定工作流、可玩 HTML 成品与秘书（产品约定）

**主线（串行 + 一段并行，与实现对齐的意图）**

1. **会议**：立项、章程草稿/归档（会议室抽屉）。
2. **制作人**：立项通过后首包入队（默认 `producer`），输出可执行任务清单。
3. **策划**：按清单落地玩法与验收点（派单给策划向 Agent）。
4. **程序 / 美术**：在规格冻结后可并行派单（仍通过队列与部门工单驱动）。
5. **QA**：集成后走 QA/部门通过或驳回。

**成品放哪、怎么打开**

- 可玩内容以 **单页 HTML** 形式保存到仓库：`production/preview/<projectId>/index.html`（Studio「显示器」→ 粘贴 → **保存到显示器**）。
- 浏览器直接打开：`http://<studio-server 主机>:8787/preview?projectId=<projectId>`（与左侧 iframe 同源服务）。
- Studio 内：**显示器** 抽屉里 iframe 即预览；也可在资源管理器中打开上述路径文件（需允许本地脚本时注意安全）。

**秘书**

- 左栏 **秘书** 条 + 菜单 **通知**：提示流程、卡点（如 WS 断开、长时间生成/排队、尚无预览文件）、以及保存预览/任务成功后的 **URL 与磁盘路径**。
- 自动化不验秘书文案；手测确认与当前操作一致即可。

**默认预览（占位页）**

- `production/preview/project_1/index.html` 为仓库内置**占位说明**，用于联调显示器路径；**不是**会议或队列自动生成。程序任务落盘或手动保存新 HTML 后即被替换。

**制作人自动衔接**

- 策略 → 制作人：**自动派单** 开启时，**立项首包（meeting_kickoff）任务成功结束后** 会自动再入队策划→程序→美术→QA 四条衔接任务（随队列与并发槽执行）。关闭后则仅首包，部门工单「通过」仍可手动派活。
